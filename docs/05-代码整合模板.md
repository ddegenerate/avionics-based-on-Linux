# 代码整合模板

前面各分区的程序分散在 `avionics_sensor.c`、`avionics_sche.c`、`avionics_mem.c` 三个文件里，每次换方案都得重新编译另一个。这里把它们合并成一个统一模板，通过文件头部两个 `#define` 开关控制启用哪些功能。

---

## 设计思路

四个版本的核心差异就两个：

| 开关 | 作用 | 对应原文件 |
| :--- | :--- | :--- |
| `USE_TIME_PARTITION` | 开启 `SCHED_DEADLINE` 时间窗口 | `avionics_sche.c` |
| `USE_CMA_MEM` | 开启 CMA 连续物理内存 | `avionics_mem.c` |

选用编译期开关而非运行时 `getopt` 传参，原因是功能选定后编译器自动裁掉无用分支，零运行时开销，也更接近真实嵌入式系统的构建方式。CPU 隔离和 L3 缓存分区属于系统级配置，只需把编译好的程序放进对应的 cgroup/resctrl 资源组运行即可，代码无需感知。

---

## `avionics_unified.c`

完整代码见 `src/avionics_unified.c`，结构如下：

```c
#define _GNU_SOURCE
#include <stdio.h>
#include <stdlib.h>
#include <stdint.h>
#include <time.h>

// ==================== 功能开关 ====================
// 需要哪个功能就把哪行注释去掉（或编译时加 -D 定义）

// #define USE_TIME_PARTITION    // SCHED_DEADLINE 时间分区
// #define USE_CMA_MEM           // CMA 连续物理内存分区

// ==================== 时间分区相关 ====================
#ifdef USE_TIME_PARTITION
#include <unistd.h>
#include <sys/syscall.h>
#include <linux/sched.h>

// ... sched_attr 结构体与 sched_setattr 封装 ...
// ... setup_time_partition() 函数 ...
#endif

// ==================== 内存分区相关 ====================
#ifdef USE_CMA_MEM
#include <fcntl.h>
#include <string.h>
#include <sys/mman.h>

// ... setup_cma_memory() / teardown_cma_memory() ...
#endif

// ==================== 核心计算逻辑 ====================
// ... matrix_multiply() ...

int main() {
#ifdef USE_TIME_PARTITION
    setup_time_partition();
#endif

    // 根据开关选择内存来源
#ifdef USE_CMA_MEM
    cma_base = setup_cma_memory();
    // 在 CMA 映射区上切分 A、B、C 矩阵
#else
    A = malloc(...); B = malloc(...); C = malloc(...);
#endif

    // 填充数据 → 预热 → 计时测试 → 输出结果 → 清理
}
```

## 编译与使用

```bash
# 不开任何分区（等价于 avionics_sensor）
gcc -O0 avionics_unified.c -o avionics_unified

# 只开时间分区（等价于 avionics_sche）
gcc -O0 -DUSE_TIME_PARTITION avionics_unified.c -o avionics_unified

# 只开内存分区（等价于 avionics_mem）
gcc -O0 -DUSE_CMA_MEM avionics_unified.c -o avionics_unified

# 全开
gcc -O0 -DUSE_TIME_PARTITION -DUSE_CMA_MEM avionics_unified.c -o avionics_unified
```

运行（始终需要 sudo 和正确的系统环境）：

```bash
sudo ./avionics_unified
```

> **前置条件：** 时间分区要求终端已在 cgroups 分区内（`/sys/fs/cgroup/avionics_partition`），CMA 要求 `avionics_cma.ko` 已加载。缓存分区要求在 resctrl 资源组内（`/sys/fs/resctrl/avionics_cache`），但这个是系统级配置，代码无感知。

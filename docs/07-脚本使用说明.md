# 脚本与参数修改说明

项目提供了四个自动化脚本，对应四层分区的配置。所有脚本均需 `sudo` 运行。

---

## 1. setup_core.sh —— CPU 核心隔离

**作用：** 创建 cgroups v2 子 cgroup，将指定核心从全局调度器中独立出来。

**可修改参数：**

| 参数 | 位置 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `PARTITION_NAME` | 第 12 行 | `avionics_partition` | cgroup 子目录名，可以改成其他名称 |
| `ISOLATED_CPUS` | 第 15 行 | `10-11` | 要隔离的核心范围 |

**换核心怎么做：**

```bash


# 1. 先查目标逻辑核的物理兄弟核
cat /sys/devices/system/cpu/cpu<目标核>/topology/thread_siblings_list
# 假设查 cpu7 的兄弟核，输出 6-7

# 2. 把脚本第 15 行改成查到的范围
ISOLATED_CPUS="6-7"
```

**配套改动：** 需要同步修改 GRUB 中的 `nohz_full` 和 `rcu_nocbs` 参数，以及 `setup_cache.sh` 中绑定的核心范围。

**GRUB 参数位置：** `/etc/default/grub`，在 `GRUB_CMDLINE_LINUX_DEFAULT` 中修改：
```
nohz_full=6,7 rcu_nocbs=6,7
```

**运行：**
```bash
sudo ./setup_core.sh
# 脚本结束后手动执行（把终端放入分区）：
sudo sh -c "echo $$ > /sys/fs/cgroup/avionics_partition/cgroup.procs"
```

## 2. setup_memory.sh —— CMA 内存分区

**作用：** 编译并加载 CMA 内核驱动，创建 `/dev/avionics_cma` 设备节点。

**脚本本身无需修改。** 它自动定位 `src/cma_avionics/` 目录并编译。

**修改内存大小的正确流程（三步联动）：**

| 步骤 | 位置 | 参数 | 说明 |
| :--- | :--- | :--- | :--- |
| GRUB | `/etc/default/grub` | `cma=520M` | 预留空间，需略大于实际需求（512M → 设 520M） |
| 内核驱动 | `src/cma_avionics/avionics_cma.c` 第 7 行 | `CMA_MEM_SIZE` | 实际申请的连续内存，单位是字节 |
| 用户态程序 | `src/avionics_mem.c` 或 `src/avionics_unified.c` | `TOTAL_SIZE` | 必须与驱动的 `CMA_MEM_SIZE` 完全一致 |

假设把 512MB 改成 256MB：

```bash
# GRUB: cma=264M     (256 + 8 余量)
# 驱动: #define CMA_MEM_SIZE (256 * 1024 * 1024)
# 用户态: #define TOTAL_SIZE (256 * 1024 * 1024)
```

> 改了 GRUB 参数后必须 `sudo update-grub && sudo reboot`。

**运行：**
```bash
sudo ./setup_memory.sh
```

## 3. setup_cache.sh —— L3 缓存分区

**作用：** 挂载 resctrl、修改全局掩码、创建航电专属资源组并设为独占、绑定核心。

**可修改参数（均在脚本中硬编码，需直接改脚本）：**

| 参数 | 行号 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| 全局掩码 | 第 38 行 | `L3:0=fff0;1=fff0` | 腾出尾部的 4 个 bit |
| 分区掩码 | 第 53 行 | `L3:0=f;1=0` | 分配 `f`（4 bit = 2MB）给航电 |
| 绑定核心 | 第 67 行 | `10-11` | 哪些核心使用这个缓存分区 |

**掩码计算方法：**

`cbm_mask` 显示 `ffff` = 16 bit，每 bit 控制 `8MB / 16 = 512KB`。

| 掩码值 | bit 数 | 对应缓存 |
| :--- | :--- | :--- |
| `f`（`1111`） | 4 bit | 2 MB |
| `7`（`0111`） | 3 bit | 1.5 MB |
| `3`（`0011`） | 2 bit | 1 MB |
| `1`（`0001`） | 1 bit | 512 KB |
| `ff`（`1111 1111`） | 8 bit | 4 MB |

**换核心或改缓存大小怎么做：**

比如要把核心 6-7 放进分区，缓存改成 1.5MB（3 bit = `7`）：

1. 全局掩码改为 `L3:0=fff8;1=fff8`（腾出尾部 3 个 bit）
2. 分区掩码改为 `L3:0=7;1=0`
3. 绑定核心改为 `6-7`
4. `setup_core.sh` 那边也要同步改成 `ISOLATED_CPUS="6-7"`

> **注意：** Ryzen 5500U 有两个 CCX，对应的 `L3:0` 和 `L3:1` 掩码必须同时退让，否则内核报 `Schemata overlaps`。如果你的目标核心在 CCX 1 上，换过来就行（`L3:0=0;1=f`）。

**运行：**
```bash
sudo ./setup_cache.sh
```

## 4. run_stress.sh —— 压力测试

**作用：** 封装 `stress-ng`，在非隔离核心上制造 CPU、缓存、内存压力。

**可修改参数：**

| 参数 | 行号 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `CPU_WORKERS` | 第 6 行 | `10` | CPU 运算线程数，设为非隔离逻辑核数 |
| `CACHE_WORKERS` | 第 8 行 | `2` | 缓存冲击线程数，按 cache line 粒度遍历填充 L3 |
| `VM_WORKERS` | 第 10 行 | `2` | 内存读写线程数，申请/释放大量内存 |
| `TIMEOUT` | 第 12 行 | `60s` | 持续时间 |

**参数选择建议：**

```
CPU_WORKERS  ≈ 总逻辑核数 - 已隔离逻辑核数
CACHE_WORKERS ≈ 2~4（多了也不会线性增加压力，瓶颈在 L3 带宽）
VM_WORKERS    ≈ 2~4（每个线程默认反复读写 256MB，按需调整）
```

如果你隔离了核心 10-11（2 个逻辑核），12 - 2 = 10，所以 `CPU_WORKERS=10`。如果把核心 6-7 也隔离了，就改成 8。

**运行：**
```bash
sudo ./run_stress.sh
```

## 5. C 代码中需要与脚本联动的参数

### 时间窗口参数

在 `src/avionics_sche.c` 或 `src/avionics_unified.c`（开启 `USE_TIME_PARTITION` 时）：

```c
#define SCHED_RUNTIME_MS  3    // 每周期运行时间
#define SCHED_DEADLINE_MS 10   // 截止期限
#define SCHED_PERIOD_MS   10   // 周期
```

三者关系：`Runtime ≤ Deadline ≤ Period`。一般设 `Deadline = Period`。

### 测试规模参数

在靶标程序（`avionics_target.c` / `avionics_unified.c` 中）：

```c
#define NUM_ITERATIONS 10000   // 测试循环次数
#define MATRIX_SIZE 50         // 矩阵边长，计算量 = SIZE³
```

`MATRIX_SIZE=50` 时单次矩阵乘法约 50³ = 125,000 次浮点运算。改大或改小会直接影响单次耗时，`SCHED_RUNTIME_MS` 需要跟着调——Runtime 必须大于 WCET，否则程序永远跑不完一个窗口就被挂起。

## 参数联动速查表

改一个参数时，需要同时改哪些地方：

| 改什么 | 涉及文件/位置 |
| :--- | :--- |
| 换隔离核心 | `setup_core.sh` → `ISOLATED_CPUS`、GRUB → `nohz_full`/`rcu_nocbs`、`setup_cache.sh` → 绑定核心 |
| 改 CMA 内存大小 | GRUB → `cma=`、`avionics_cma.c` → `CMA_MEM_SIZE`、`avionics_unified.c` → `TOTAL_SIZE` |
| 改 L3 缓存分配量 | `setup_cache.sh` → 全局掩码 + 分区掩码 |
| 改时间窗口 | `avionics_unified.c` → `SCHED_RUNTIME_MS`/`SCHED_DEADLINE_MS`/`SCHED_PERIOD_MS` |
| 改测试规模 | `avionics_unified.c` → `NUM_ITERATIONS`/`MATRIX_SIZE`，可能需要联动调整 `SCHED_RUNTIME_MS` |
